## 1. 產品定義

### 1.1 一句話

> 透過每天完成幾乎不可能失敗的微小挑戰，累積真實世界的成功經驗，並把這些經驗化為建設一座專屬王國的力量。

玩家得到的不是等級、戰力、裝備，而是 **自我效能感（Self-Efficacy）**：「我做得到 / 我比以前更勇敢 / 我真的改變了。」

### 1.2 受眾

曾經過度設定目標、失敗、失去動力的人。產品的真正任務是 **恢復「felt competence（被自己感覺到的能力）」**，習慣養成是自然的下游結果，不是被明說的目標。

### 1.3 五大設計支柱

1. 保證 >90% 成功率
2. 「永不失敗」：沒登入的日子＝角色在休息，不是失敗
3. 可視化的漸進成長（WYSIWYG）
4. 溫柔的第二人稱語氣
5. 比較故事，而非比較分數

### 1.4 不可妥協的鐵則（NEVER-FAIL INVARIANTS）— 程式必須結構性保證

這些不是語氣偏好，是 **結構性約束**，要在資料模型與邏輯層強制，不能只靠文案：

- **不存在任何會「斷」的連續計數（streak）。**
- **任何王國元素一旦出現，永不被移除、永不變暗、永不衰退。** 沒有「荒廢 / 枯萎 / 流失」。
- **沒登入時**畫面顯示「島嶼靜謐運轉中」，永遠不顯示「你失敗了 / 你已經 N 天沒來」。
- **不得有任何可能長期填不滿的進度條**（尤其禁止全域目標條，見第 11.3）。
- **每日卡永遠可完成**：安全卡保證是送分題。
- **沒有 per-player 排行榜或分數。** 任何社群計數一律 per-challenge（見第 11.2）。
- 王國只會「增加」，永不要求玩家「補回損失」（世界觀是甦醒，不是修復，見第 2.1）。

> 實作守門：任何新功能在合併前，先對照這 7 條。違反任一條的功能不進 MVP。

## 2. 美術與世界觀方向

### 2.1 甦醒的世界，不是壞掉的世界

王國是 **一片安靜、等待被點亮的土地**，不是一座你有責任修好的廢墟。每個動作都是純粹的「多」，不是「補回過去的錯」。背景敘事可以叫「王國」，但 **不採用「文明曾因人們不再相信自己而碎裂、你來重建」這種帶愧疚的修復框架**。

### 2.2 視覺風格

- **像素藝術 + 發光**，深色星空背景，一座漂浮在星空中的島嶼王國。
- **渲染用 Canvas 2D**。**明確放棄**先前文件中的 3D Billboard / 偽 3D 方案——對這個產品是過度設計。
- 元素全部「會發光」，呼應「點亮、光不熄滅」。
- 表面語氣輕鬆搞笑，底層是療癒（「表面搞笑，底層療癒」）。陪伴角色 **阿光**：日常搞笑、里程碑時刻真誠（見第 12 節）。

## 3. 核心系統：屬性與王國元素

### 3.1 五大人格屬性

| key | 中文 | 意義 | 顏色（glow） | 現實動作範例 |
|---|---|---|---|---|
| `courage` | 勇氣 | 面對未知 | `#FFB454` | 主動打招呼、表達意見、嘗試新事物 |
| `vitality` | 活力 | 身體與行動 | `#79C8FF` | 散步五分鐘、伸展、提早起床 |
| `focus` | 專注 | 持續完成 | `#6E9BFF` | 專心讀五分鐘、整理桌面、完成待辦 |
| `curiosity` | 好奇 | 探索世界 | `#B08CFF` | 查不懂的知識、走不同路、學新技能 |
| `warmth` | 溫暖 | 建立連結 | `#FF8A6B` | 感謝一個人、關心朋友、稱讚他人 |

### 3.2 屬性 → 王國元素 mapping（核心）

**完成一張屬性卡，就在王國加一個對應元素。** 不需要中間「碎片累積到門檻才長」的機制——MVP 中「完成 1 張 = 加 1 元素」，計數與元素 1:1。

| 屬性 | 王國元素 | 行為 |
|---|---|---|
| 勇氣 | **一格新土地** | 王國領土往外擴張一格 |
| 活力 | **一位居民** | 出現一個會在島上走動的小人 |
| 專注 | **一座光塔** | 立新塔，或把現有塔加高一層 |
| 溫暖 | **一間小屋** | 在土地上蓋一間發光小屋 |
| 好奇 | **一棵樹** | 種一棵發光的樹 |

> 設計註記：勇氣＝開土地，同時解決了「單一物件無限脹大」與「往星空伸出去的橋很怪」兩個舊問題，也最貼「勇氣＝拓展自己的邊界」。

### 3.3 成長規則（每元素如何生成）

座標系為方格 `(c, r)`，`c` 為欄、`r` 為列，原點在島中心。

- **勇氣 / 加土地**：從現有任一土地格隨機挑一個，往四鄰隨機方向找一個「界內且尚未是土地」的格，加入 `land`。界限 `|c| <= 6 && -4 <= r <= 4`（可調）。
- **溫暖 / 小屋**：挑一個「無建築佔用」的土地格放小屋；若無空格，退而選任一土地格。佔用記入 `occupied`。
- **好奇 / 樹**：挑任一土地格放樹（可密集，形成林）。樹不佔用 `occupied`。
- **專注 / 光塔**：若已有未滿高的塔（上限 `h=6`），50% 機率挑一座 `h++`；否則在空格立新塔 `h=1`。
- **活力 / 居民**：`citizenCount++`；在隨機土地格生成一個會走動的小人（位置為 runtime，不入庫，見第 7.4）。

所有操作 **只新增、不刪除**。

## 4. 每日迴圈（Daily Loop）

每天 30 秒～2 分鐘。

### 4.1 三張卡——以「角色」區分，三張都帶屬性、都加元素

| 卡 | 角色定位 | 屬性來源 |
|---|---|---|
| **安全卡 safe** | 守住 >90% 成功率，是送分題 | 隨機五屬性之一，動作極易完成 |
| **主線卡 main** | 對齊玩家 onboarding 選的方向，是個人化主槓桿 | = 玩家選的方向屬性（見 4.4） |
| **驚喜卡 surprise** | 新鮮感與傳播，可荒誕、可換 | 隨機屬性；可換一次 |

> 註：先前曾有「只有屬性卡提升能力」的舊版設定，**已作廢**。最終定案是三張卡都帶屬性、都加元素（連送分的安全卡也推進成長）。

### 4.2 抽卡演算法（每日決定，可重入）

當天三張卡需 **對同一使用者同一天穩定**（重開 App 看到同樣三張、保留完成狀態）。用 `seed = hash(userId + 'YYYY-MM-DD')` 的 seeded RNG。

```
function drawDailyCards(userId, date, direction):
    rng = seededRNG(hash(userId + date))
    safe     = pickFrom(pool.safe, rng)                       // 隨機屬性
    main     = pickFrom(pool.main.filter(attr == direction), rng)
                 // 20% 機率允許 off-direction，增加變化（可調 MAIN_OFF_DIRECTION_RATE）
    surprise = pickFrom(pool.surprise, rng)                   // 隨機屬性，可換
    return {
      date,
      cards: [ tag(safe,'safe'), tag(main,'main'), tag(surprise,'surprise', swappable=true) ],
      completed: { safe:false, main:false, surprise:false },
      swapsUsed: 0
    }
```

換卡：`surprise` 可換 `SWAPS_PER_DAY`（預設 1）次，從 surprise pool 另抽一張。

### 4.3 完成處理

```
function completeCard(slot):
    if state.daily.completed[slot]: return            // 不可重複
    card = state.daily.cards[slot]
    state.daily.completed[slot] = true
    state.counts[card.attribute] += 1
    addElement(card.attribute)                         // 見 3.3
    appendCodex({ date: today, attribute: card.attribute, text: card.text })
    persistLocal(); queueSync()
    playFeedback(card.attribute)                       // 完成動畫 + 阿光可能說一句
```

**沒有「取消完成」、沒有任何懲罰、沒有逾時失效。** 未完成的卡不累積負面狀態，只是隔天換新卡。

### 4.4 Onboarding 方向 → 屬性

| 方向選項 | 對應屬性 |
|---|---|
| 動起來 | `vitality` |
| 讀點書 | `focus` |
| 勇敢一點 | `courage` |
| 對人好一點 | `warmth` |
| 多看看世界 | `curiosity` |

主線卡固定餵這個方向，安全卡與驚喜卡隨機散在五屬性。結果：玩家選的方向長最快，島嶼自然是「他的形狀」。

## 5. 卡片資料模型

### 5.1 Card schema（JSON）

```json
{
  "id": "warmth_thanks_001",
  "attribute": "warmth",
  "role": "safe | main | surprise",
  "text": "今天跟一個人說聲謝謝",
  "tone": "gentle | absurd",
  "difficulty": 1
}
```

### 5.2 三個卡池

- **安全卡池 safe**：**官方策展，永不開放玩家創作**。動作極易、保證可完成。
- **主線卡池 main**：官方策展，依屬性分類。
- **驚喜卡池 surprise**：MVP 為官方策展（含荒誕梗）。MVP+（選配）可加入「玩家畫給自己」的卡（見第 11.1）。Phase 2 才接入「別人創作」。

> MVP 卡池以 **靜態 JSON 檔** 形式放在 `/public/cards/{safe,main,surprise}.json`，由 service worker 快取。**不需要為策展卡建資料表**，省成本也省頻寬。

### 5.3 範例卡（每屬性各舉一）

| 屬性 | 安全卡 | 主線卡 | 驚喜卡（搞笑） |
|---|---|---|---|
| 勇氣 | 對著鏡子說一句「我可以」 | 在群組裡先發第一則訊息 | 走進房間時說「朕來了」 |
| 活力 | 站起來伸個懶腰 | 出門散步五分鐘 | 假裝自己是企鵝走三步 |
| 專注 | 把桌上一樣東西歸位 | 專心讀五分鐘不看手機 | 回答任何問題前先說「根據資料顯示」 |
| 好奇 | 查一個你今天好奇的字 | 走一條沒走過的路回家 | 跟家裡的垃圾桶自我介紹 |
| 溫暖 | 對一個人說謝謝 | 主動關心一位朋友 | 對著冰箱說「辛苦了」 |

## 6. 信心圖鑑（Confidence Codex）— 產品的心臟

每完成一張卡就留下一條紀錄。玩家回顧時感到「原來我已經做到這麼多」。**這是 MVP 不可妥協的功能。**

### 6.1 Entry schema

```json
{ "id": 1024, "date": "2026-06-23", "attribute": "courage", "text": "在群組裡先發第一則訊息" }
```

### 6.2 行為

- 反向時間排序（最新在上）。
- 顯示：第幾天 / 日期 · 屬性色點 · 動作文字。
- 頂部彙總：「已記下 N 件做到的事」。
- 只增不刪。可分頁載入。

> Phase 2 會在此之上長出「未來自己 / 回顧」這個情緒錨點功能；MVP 先做樸素的清單。

## 7. 王國渲染模型（Canvas 2D 實作細節）

以下常數來自已驗證的原型，可直接採用。

### 7.1 座標與常數

```js
const W = 400;          // canvas 邏輯尺寸（CSS 再響應式縮放，image-rendering: pixelated）
const TS = 18;          // tile size (px)
const H2 = 9;           // half tile
const OX = 200, OY = 212; // 島中心原點（偏下，讓塔有往上長的空間）
const q  = 2;           // sprite 像素塊邊長

const cx = c => OX + c * TS;   // tile 中心 x
const cy = r => OY + r * TS;   // tile 中心 y
// tile 左上角 = (cx(c)-H2, cy(r)-H2)
// sprite 底部錨點（站在格上）= (cx(c), cy(r)+7)
```

色票：

```js
const COL = {
  courage:'#FFB454', vitality:'#79C8FF', curiosity:'#B08CFF',
  warmth:'#FF8A6B', focus:'#6E9BFF'
};
// 土地：top #34715f / 高光 #498c76 / 邊框 #24514a / 土壤側面 #3a2c22
// 背景星空：#070a14
```

### 7.2 每幀繪製順序

1. 清畫面，填星空背景，畫會閃爍的星點。
2. 島嶼底部柔光（一個 radial gradient，以土地質心為中心）。
3. **土地格**：依 `(row, col)` 排序繪製；若某格下方無土地，於其底緣畫數 px 土壤側面，形成漂浮厚度感；土地格畫頂面 + 頂部高光線 + 邊框。
4. **實體**（小屋／樹／塔／居民）收集成陣列，依「底部 y」升序排序後繪製，確保前後遮擋正確。
5. **完成脈衝**：擴散淡出的光圈（reduced-motion 時略過）。

### 7.3 Sprite 定義（像素網格，glow 用 shadowBlur）

每個 sprite 是字元網格，非 `.` 的格畫成 `q×q` 方塊；標記為 glow 的字元在填色前設 `shadowBlur`。

```js
// 小屋（溫暖）— 屋頂 r / 屋身 b / 發光窗 w / 門 d；w 發光
const HOUSE = ["...rrrrr...","..rrrrrrr..",".rrrrrrrrr.","rrrrrrrrrrr",
  ".bbbbbbbbb.",".bbbwwbbbb.",".bbbwwbbbb.",".bbbbbbbbb.",".bbbdddbbb.",".bbbdddbbb."];

// 樹（好奇）— 樹冠 c/C 發光、樹幹 t
const TREE = ["...ccc...","..ccccc..",".ccCCCcc.",".ccCCCcc.","ccCCCCCcc",
  "ccCCCCCcc",".ccCCCcc.","..ccccc..","...ttt...","...ttt...","...ttt..."];

// 居民（活力）軀幹 + 兩種腿部 frame；頭 k 發光、衣 c（每人隨機色）、腿 l
const PBODY = [".kk.",".kk.","cccc","cccc","cccc"];
//  站立腿 ["l..l","l..l"]，走路另一 frame [".ll.",".ll."]
```

- **光塔（專注）** 程序化繪製：寬 `7q` 的塔身（`#3b4670`），沿途每隔幾格畫發光窗（`#8fb0ff`），頂部畫三段發光雉堞（`COL.focus`）。高度 `= h * 4q`，`h` 為塔的層數。

### 7.4 居民移動（直接回答「人能不能動」：能，且極便宜）

- 每位居民 `{ x, y, tx, ty, moving, pal }`，位置為 runtime，不入庫。
- 每幀：朝目標 `(tx,ty)` 移動 `~0.35 px`；到達後挑「隨機土地格中心 ± 小偏移」作為新目標。
- 動畫：移動時兩腿 frame 交替 + 上下 1px bob。**無尋路、無物理、無碰撞。**
- 開 App 時依 `citizenCount` 生成對應數量居民並各給隨機初始目標。
- reduced-motion 時居民靜止站立。

## 8. 資料架構

### 8.1 本地狀態 shape（IndexedDB，local-first 的真相來源）

```json
{
  "version": 1,
  "direction": "courage",
  "counts": { "courage": 0, "vitality": 0, "focus": 0, "warmth": 0, "curiosity": 0 },
  "land":   [[0,0],[1,0],[0,1]],
  "houses": [[0,1]],
  "trees":  [[1,0]],
  "towers": [[0,-1,3]],
  "citizenCount": 4,
  "firstDay": "2026-06-23",
  "lastActive": "2026-06-25"
}
```

> 整份狀態極小（即使重度玩家累積上千元素也僅數十 KB）。畫面由這份狀態在本地畫出，同步時只傳這份資料。

### 8.2 Supabase schema（PostgreSQL）

```sql
-- 使用者基本資料（方向、暱稱）
create table profiles (
  user_id    uuid primary key references auth.users(id),
  display_name text,
  direction  text check (direction in
                ('courage','vitality','focus','warmth','curiosity')),
  created_at timestamptz default now()
);

-- 王國狀態：一人一列，整包 jsonb（成本最低）
create table kingdoms (
  user_id    uuid primary key references auth.users(id),
  state      jsonb not null,
  updated_at timestamptz default now()
);

-- 信心圖鑑：成長核心，獨立成表以利分頁與回顧
create table codex_entries (
  id         bigserial primary key,
  user_id    uuid references auth.users(id),
  date       date not null,
  attribute  text not null,
  text       text not null,
  created_at timestamptz default now()
);
create index codex_user_date on codex_entries (user_id, created_at desc);

-- 當日卡片與完成狀態（可重入）
create table daily_cards (
  user_id    uuid references auth.users(id),
  date       date not null,
  data       jsonb not null,   -- { cards:[...], completed:{...}, swapsUsed }
  primary key (user_id, date)
);
```

- 啟用 **Row Level Security**：每張表 `user_id = auth.uid()` 才可讀寫。
- 策展卡池 **不入庫**，走靜態 JSON（見 5.2）。

### 8.3 同步策略（local-first）

1. 開 App **先讀 IndexedDB**，立即出畫面（幾乎零等待、零頻寬）。
2. 背景向 Supabase 拉取 `updated_at` 較新者；單人資料，衝突採 **last-write-wins**。
3. 完成卡片即時寫本地，加入同步佇列，連線時批次 upsert。
4. 全程可離線操作，恢復連線再同步。

### 8.4 靜態卡池

`/public/cards/safe.json`、`main.json`、`surprise.json`，service worker 首次載入後永久快取；更新卡池＝改檔重新部署。

## 9. 技術架構

### 9.1 Stack

- **前端**：原生 JS + Canvas 2D（或薄框架），PWA。無重型 3D／遊戲引擎。
- **託管**：Vercel。
- **後端 / DB / Auth**：Supabase。
- **版控 / CI**：GitHub。

### 9.2 PWA / 離線

- `manifest.json` + service worker。
- 快取殼層（HTML/JS/CSS）、sprite 圖（若改用圖集）、卡池 JSON。
- 安裝後每次開啟幾乎不耗頻寬：畫面本地繪製，只同步小狀態。

### 9.3 Auth

- Supabase Auth。MVP 建議 **匿名 / 裝置綁定** 或 **magic link**，降低 onboarding 摩擦（也為 Phase 2 的「免註冊外部分享連結」鋪路）。
- 之後可升級為 email / OAuth 帳號以跨裝置。

> 安全註記：帳號建立、密碼輸入由使用者本人在官方流程完成，不由自動化代填。

## 10. App 結構與畫面（MVP）

1. **Onboarding**：阿光打招呼 → 選一個方向（5 選 1）→ 種下第一格土地。
2. **首頁 Home**：上方今日三張卡 + 完成鈕；下方即時王國視圖（會走動的居民）。完成卡片時就地長出元素 + 脈衝回饋。
3. **信心圖鑑 Codex**：回顧清單。
4. **設定 Settings**：方向、音效、reduced-motion、帳號。

> 首頁把「卡片」與「王國」放同一畫面（如原型），減少跳轉，強化「我做了 → 世界長了」的因果即時性。

## 11. 範圍分層（明確切線）

### 11.1 MVP（Phase 1）— 現在要做

- Onboarding + 方向
- 每日三卡（策展靜態卡池）+ 換驚喜卡
- 完成 → 王國長元素（五元素 + 會走動的居民）
- 信心圖鑑
- never-fail 鐵則（結構性）
- local-first + Supabase 同步、PWA、離線
- **MVP+（選配，低風險）**：玩家用繪圖法 **畫卡給自己**，進自己的驚喜卡池。**僅限自己玩、不分享、無人數、無門檻**，因此不需審核。

### 11.2 Phase 2 — 有一群活躍玩家後

- **UGC 分享**：玩家創作卡 → 送到其他真人面前。**必經五階安全管線**（進度解鎖創作 → 自動過濾 → 人工/社群審核 → 限量試水池 → 公開池含檢舉下架）。**安全卡池永不開放創作。**
- **卡片 → 建築進化**：用 **per-challenge（每張卡）完成人數**累積成更大建築。**嚴禁 per-player 計數**（會引發排行榜毒性）。
- 繪圖畫布隨「該卡被玩次數」解鎖更大格（3×3 → 5×5）。
- 「達門檻才能上架到別人能看到的池」。
- 「精準試水溫」：新卡先靜默派發給少數屬性相符玩家；冷門則自然消散，無被拒挫折感。
- 外部免註冊分享連結（IG / LINE），內外完成數分開記錄。
- 「雙向飛行的光芒」：完成他人卡時，光一分為二（自己 + 出題者）。
- **信心圖鑑 → 未來自己 / 回顧**情緒錨點。
- 故事 / 區域敘事層（Story Bible，五區互聯元敘事）。可作為「玩家以為在玩搞笑收集，後段揭曉這座王國一直叫『心之國度』，收集到的其實是被自己找回的部分」這個伏筆。
- 阿光語庫擴充（預寫輪播，避免每日生成的內容跑步機）。

### 11.3 Phase 3+ — 規模夠大才談

- 世界文明 / 全域紀元（全球月度累積解鎖）。**注意：全域目標條在玩家數不足時會永遠填不滿＝全員可見的失敗，違反鐵則；務必等活躍量足夠且設計成「永遠在前進、無法落後」才可上。**
- 付費分層（月費：更多卡槽、王國外觀、特效、紀錄分析；單次：紀念建築、故事館、造型包）。

> 一句話：別讓「信心王國」這個願景，吃掉「信心王國」這個做得出來的東西。

## 12. 文案與語氣

- **第二人稱、溫柔、表面搞笑**。
- **阿光**：日常搞笑、里程碑真誠。語句採 **預寫輪播語庫**，不做每日即時生成（避免長期內容負擔）。
- 失敗情境一律改寫為休息：「島嶼靜謐運轉中」「你的光還在，只是今天先歇著」。
- 里程碑（第 7 天、第 30 天、某屬性首次達 N）觸發阿光的真誠台詞 + 王國小慶祝。

## 14. 建議建置順序（Milestones）

1. **M1 渲染骨架**：Canvas 王國（土地 + 五元素 + 走動居民），用假資料跑通（原型已驗證）。
